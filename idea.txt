# CYBERCORE CSS Framework — Complete Specification Document

## Project Vision

CYBERCORE is a **pure CSS design system** inspired by the aesthetics of Cyberpunk 2077 and the broader cyberpunk genre. It delivers a dark, neon-soaked, high-tech visual language that feels like hacking into Night City's neural network. The framework requires **zero JavaScript** — every effect, animation, and interaction is achieved through modern CSS alone.

The goal is to create a **2026+ optimized, framework-agnostic CSS library** that any project can drop in and use immediately, whether it's vanilla HTML, React, Vue, Svelte, Astro, or server-rendered PHP/Ruby/Python templates.

---

## Core Design Philosophy

### 1. Pure CSS, No JavaScript Dependencies
Every single feature — glitch effects, neon glows, animated borders, hover states, loading spinners, progress bars, form validation states — must work with CSS only. JavaScript can enhance but is never required.

### 2. Framework Agnostic
The library ships as plain CSS and SCSS. It uses semantic class names that work in any templating system. No JSX dependencies, no CSS-in-JS, no build tool requirements (though SCSS compilation is supported).

### 3. Modern CSS Features (2026+)
Leverage the latest CSS capabilities that now have broad browser support:
- `color-mix()` for dynamic color manipulation
- CSS custom properties (variables) for theming
- Container queries for component-level responsiveness
- `@layer` for cascade management
- Scroll-driven animations
- `:has()` selector for parent selection
- `clamp()` for fluid typography and spacing
- `oklch()` color space for perceptually uniform colors
- Nesting (native CSS nesting, not just SCSS)
- `@scope` for scoped styles
- View transitions API (CSS portion)

### 4. Performance First
- No render-blocking JavaScript
- GPU-accelerated animations using `transform` and `opacity`
- Minimal repaints and reflows
- Tree-shakeable SCSS modules
- Small footprint when minified and gzipped

### 5. Accessibility Built-In
- Respects `prefers-reduced-motion`
- Respects `prefers-color-scheme` (for potential light mode variant)
- Respects `prefers-contrast`
- Full keyboard navigation support
- Proper focus states
- ARIA-friendly markup patterns

## Color System

### Primary Palette

The framework uses four signature accent colors inspired by Cyberpunk 2077's visual language:

| Name | Primary Hex | Usage |
|------|-------------|-------|
| **Cyber Cyan** | `#00f0ff` | Primary actions, links, default accent, tech/neutral |
| **Cyber Magenta** | `#ff2a6d` | Danger, errors, alerts, aggressive/hostile |
| **Cyber Yellow** | `#fcee0a` | Warnings, highlights, caution, Militech vibe |
| **Cyber Green** | `#05ffa1` | Success, confirmation, money, Arasaka corpo vibe |

Each color has a full 9-step scale (100-900) for flexibility:
- 100-300: Light tints (for subtle backgrounds, disabled states)
- 400-500: Primary vibrant shades
- 600-900: Dark shades (for borders, shadows, depth)

### Neutral Palette

Two neutral scales:

| Name | Usage |
|------|-------|
| **Cyber Void** | Ultra-dark blacks for backgrounds (main bg is `#0d0e14`) |
| **Cyber Chrome** | Metallic grays for text and UI chrome |

### Semantic Color Tokens

Map raw colors to semantic purposes:
```
--color-bg-primary      → Main page background
--color-bg-secondary    → Card backgrounds, elevated surfaces
--color-bg-tertiary     → Input backgrounds, nested elements
--color-bg-elevated     → Modals, dropdowns, popovers

--color-text-primary    → Main body text
--color-text-secondary  → Subdued text
--color-text-muted      → Placeholder text, disabled
--color-text-accent     → Links, highlighted text

--color-border-default  → Standard borders
--color-border-accent   → Highlighted borders
--color-border-glow     → Glowing border effects
```

### Glow Effects

Pre-defined box-shadow values for neon glow effects:
```
--glow-cyan    → Multi-layer cyan glow (20px, 40px, 80px blur)
--glow-magenta → Multi-layer magenta glow
--glow-yellow  → Multi-layer yellow glow
--glow-green   → Multi-layer green glow
--glow-text-*  → Smaller glows for text-shadow
```

## Typography System

### Font Stack

Three font categories with fallbacks:

| Category | Primary Font | Fallbacks | Usage |
|----------|--------------|-----------|-------|
| **Display** | Rajdhani, Orbitron | Share Tech Mono, system-ui | Headings, buttons, labels |
| **Body** | Exo 2 | Roboto, system-ui | Paragraphs, general text |
| **Mono** | JetBrains Mono | Fira Code, SF Mono, monospace | Code, data, terminal |

### Type Scale

Fluid typography using `clamp()` with a Perfect Fourth ratio (1.333):

```
--text-xs   → ~12px (small labels, badges)
--text-sm   → ~14px (secondary text, buttons)
--text-base → ~16px (body text)
--text-lg   → ~20px (large body, card titles)
--text-xl   → ~25px (section headers)
--text-2xl  → ~31px (page section titles)
--text-3xl  → ~39px (major headings)
--text-4xl  → ~49px (hero subheadings)
--text-5xl  → ~61px (hero headlines)
```

### Typography Styles

- **Headings**: All uppercase, tight letter-spacing (0.02em), font-weight 700
- **Body**: Normal case, comfortable line-height (1.6)
- **Code**: Monospace, slight cyan tint, subtle background
- **Labels**: Uppercase, extra letter-spacing (0.1em), small size

### Special Typography Effects

**Glitch Heading**: A heading style with chromatic aberration effect using `::before` and `::after` pseudo-elements that show offset colored copies of the text with animated clip-path distortion.

## Border & Corner System

### Border Widths
```
--border-thin   → 1px
--border-medium → 2px
--border-thick  → 3px
```

### Border Radii
```
--radius-none  → 0
--radius-sm    → 2px
--radius-md    → 4px
--radius-lg    → 8px
--radius-cyber → 0 12px 0 12px (signature asymmetric cut)
--radius-cyber-alt → 12px 0 12px 0 (inverted asymmetric)
```

The **asymmetric corner cut** is a signature design element — it creates a futuristic angular look without being fully sharp or fully rounded.

## Global Effects

### Noise Overlay

A fixed full-screen SVG noise texture overlay at very low opacity (3%) using `feTurbulence` filter. Creates subtle film grain effect.

### Scanlines

A fixed full-screen repeating gradient creating horizontal scanlines at low opacity (4-5%). Simulates CRT monitor effect.

### Chromatic Aberration

Text effect that creates RGB split using offset `::before` and `::after` pseudo-elements with different hue colors.

### Glitch Animation

Keyframe animation that rapidly changes `clip-path` to create digital glitch distortion effect.

### Neon Border Animation

A `::before` pseudo-element with animated gradient that cycles through all accent colors, blurred for glow effect.

### Data Stream

Animated vertical gradient that creates illusion of data flowing through element.

## Installation & Usage

### Option 1: CDN Link
```html
<link rel="stylesheet" href="https://cdn.example.com/cybercore/1.0/cybercore.min.css">
```

### Option 2: npm Install
```bash
npm install cybercore-css
```

Then import:
```scss
// Import everything
@use 'cybercore-css';

// Or import specific modules
@use 'cybercore-css/scss/variables';
@use 'cybercore-css/scss/components/buttons';
```

### Option 3: Download CSS
Just download `cybercore.min.css` and link it in your HTML.

### Font Loading
Fonts can be loaded via Google Fonts CDN or self-hosted:
```html
<link href="https://fonts.googleapis.com/css2?family=Exo+2:wght@300;400;500;600;700&family=JetBrains+Mono:wght@400;500;700&family=Rajdhani:wght@400;500;600;700&family=Orbitron:wght@400;500;600;700;800;900&display=swap" rel="stylesheet">
```

## Theming & Customization

### Override Variables
All design tokens are CSS custom properties, so override them in your own CSS:

```css
:root {
  --cyber-cyan-500: #00ffff;     /* Customize primary cyan */
  --color-bg-primary: #000000;   /* Pure black background */
  --font-display: 'Your Font', sans-serif;
}
```

### SCSS Customization
If using SCSS, override variables before importing:

```scss
// Your overrides
$cyber-cyan-500: #00ffff;
$font-display: 'Your Font', sans-serif;

// Then import cybercore
@use 'cybercore-css' with (
  $cyber-cyan-500: $cyber-cyan-500,
  $font-display: $font-display
);
```

### Dark/Light Mode
The framework is dark-mode by default. A light mode variant could be added using:

```css
@media (prefers-color-scheme: light) {
  :root {
    --color-bg-primary: #f0f1f5;
    --color-text-primary: #0d0e14;
    /* etc */
  }
}
```

## Performance Guidelines

1. **Load only what you need** — Import specific SCSS modules instead of everything
2. **Minify in production** — Use `cybercore.min.css`
3. **Preload fonts** — Use `<link rel="preload">` for critical fonts
4. **Disable effects for performance** — Set `--noise-opacity: 0` and `--scanline-opacity: 0` if needed
5. **Respect user preferences** — Framework automatically respects `prefers-reduced-motion`

## Accessibility Checklist

- All interactive elements have visible focus states
- Color contrast meets WCAG AA standards
- Animations respect `prefers-reduced-motion: reduce`
- Form inputs have proper labels
- Modal traps focus appropriately (requires minimal JS)
- Skip links supported via `.cyber-sr-only`
